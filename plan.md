The migration to Swift Testing required some serialization of test runs because some tests deal with files on disk and change the current working directory (shared mutable global state) to work on them. I tried to handle this with `.serialized`, #sym:FileSystemAccessTestSuite as a serialized suite, @#sym:TemporaryDirectory and @#sym:WorkingDirectory. What we might actuallt want though is a `@ThreadLocal` current working directory as `URL.cwd` that can be set to a temporary directory with the test fixtures being copied there before the test runs. Please work on this, getting rid of all the manual synchronization and a trait that makes sure the temporary location is prepared and CWD is points there so that tests can run in parallel, but in isolation.
